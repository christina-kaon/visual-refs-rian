Emochi 生图模型理想态 · 三层目标

质量 / 内容 / 审美 三层划分 · 真人 + 动漫各一份 · 与离线 benchmark 维度对齐

TL;DR

  • 三层划分：质量（基础门槛）→ 内容（能力覆盖）→ 审美（风格轨道）。达成度递增。
  • 质量映射到离线 benchmark D1（角色一致性）+ D4（画面质量），有 hard penalty 阈值
  • 内容映射到 D2（剧情匹配）+ D3（trope 覆盖），需要 stratified test set 覆盖 emochi 真实 trope 分布
  • 审美映射到 D3（风格匹配），3 真人 + 3 动漫 = 6 个 sub-spec 独立 model（详细参考系见 visual style brief）
  • 训练 artifact 总数估算：1 base + 8 content LoRA + 6 aesthetics sub-spec = ~15 个

Cross-cutting 原则

三层难度递增的原因

质量易：靠 model architecture + 大 dataset + 训练算力就能达到。Stable Diffusion / Midjourney 主流模型都已 80%+ 解决。

内容中：靠 dataset 多样性 + tag 覆盖完整。但要做对 trope-specific 的小众 setting（如 "werewolf 雪夜营救" 这种很 niche 的场景）需要主动 curation，不是 web scrape 一把梭能搞定的。

审美难：靠美术总监级的 dataset 筛选 + 反锚点剔除。这是 emochi 真正的差异化 —— 通用 T2I 模型不会专门为 K-pop fan-edit candid 美学 curate dataset，所以 emochi 做得对就是壁垒。

三层 vs 离线 benchmark 维度 mapping

| 三层目标 | benchmark 维度 | 权重 | 评分方式 | |---|---|---|---| | 质量 Quality | D1 角色一致性 + D4 画面质量 | 0.30 + 0.15 | VLM 0-4 评分 + hard penalty 阈值 | | 内容 Content | D2 剧情匹配（trope 覆盖） | 0.30 | VLM 评 trope/scene 是否 match | | 审美 Aesthetics | D3 视觉吸引力（风格匹配） | 0.25 | VLM 评 + 风格 sub-spec 分类 |
评分流程：先 gate by Quality（不过门槛直接退回），然后 Content + Aesthetics 加权 composite。Aesthetics 在产品端额外做 sub-spec routing（同一个 prompt 在不同 sub-spec model 下给不同结果，A/B test 让用户选）。

评估方法（emochi-specific）

常规 T2I 评测（DreamBench++ / CLIP score）不适合 —— 测的是 prompt fidelity 和 general quality，不是 emochi 用户对 trope 的 emotional response。

建议 emochi-specific eval：
1. 质量：n=30 人工 calibration 实测 (李承燕 2026-04-27)，per-dim VLM Pearson D1 0.28 / D2 0.23 / D3 0.34 / D4 ~0（4 个维度 single-model 都不到 reliable 水平）。hard penalty ensemble 能把 composite 拉到 0.43。剩 70 条标完后重新跑，并且需要按 binary observable feature 重写 D4 prompt
2. 内容：按 trope × scene × shot_angle 三维 stratified sample 50-100 条/维度，确保每个组合至少 5 张 representative
3. 审美：每个 sub-spec 配 30-50 个 representative scenario，给目标用户群盲选（不能用 VLM 评，VLM 没有 K-pop fan 的 aesthetic sense）

训练流程的层级 implication

质量层：上 base model architecture + 通用大 dataset。SDXL / Flux 等开源基础足够。

内容层：在 base 上做 trope-specific LoRA fine-tuning。每个 trope category 准备 200-500 张精选样本。

审美层：每个 sub-spec 独立训练 base / 独立 LoRA，不能共享。每个 sub-spec 需要美术总监级的 dataset curation（200-1000 张精选 reference），同时主动剔除反锚点样本。

所以训练 cost 估算：base 1 个 + content LoRA × 8（每 trope 一个） + aesthetics × 6（3 真人 + 3 动漫 sub-spec）= 共 15 个模型 artifact。资源调配要按这个数算。
